## 1. 传感器应用 (sensor_app)

### 1.1 功能概述

传感器应用显示 SH3001（6 轴 IMU：加速度 + 陀螺仪）和 ST480MC（3 轴磁力计）的实时数据，并通过 Mahony AHRS 算法进行姿态解算，显示俯仰角（Pitch）、横滚角（Roll）和航向角（Yaw）。

实现位于 [`User/APP/sensor_app/sensor_app.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/sensor_app/sensor_app.c) 和 [`User/APP/sensor_app/sensor_app.h`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/sensor_app/sensor_app.h)。

### 1.3 数据结构

```c
static struct
{
    uint8_t initialized;
    uint8_t is_visible;
    short acc_data[3];       /* 加速度 X/Y/Z */
    short gyro_data[3];      /* 陀螺仪 X/Y/Z */
    short mag_data[3];       /* 磁力计 X/Y/Z */
    float temperature;       /* 温度 */
    eulerian_angles_t angles;/* 姿态角 */
} g_sensor_state;
```

### 1.4 UI 组件布局

使用 5 个 `ui_data_row_group` 组件：

```c
static ui_data_row_group_t s_temp_grp;      /* 温度 */
static ui_data_row_group_t s_attitude_grp;  /* 姿态角 */
static ui_data_row_group_t s_accel_grp;     /* 加速度计 */
static ui_data_row_group_t s_gyro_grp;      /* 陀螺仪 */
static ui_data_row_group_t s_mag_grp;       /* 磁力计 */
```

组件采用**链式布局**，每个分组的 Y 坐标由上一个分组的底部自动推算：

```c
y_pos = ui_data_row_group_get_bottom(&s_temp_grp) + 6;
s_attitude_title_y = y_pos;
y_pos = y_pos + 16 + 2;
ui_data_row_group_init(&s_attitude_grp, 20, y_pos, ...);
```

### 1.5 初始化策略

```c
static uint8_t sensor_app_init(void)
{
    titlebar_init();
    sensor_ui_components_init();

    /*
     * SH3001/IMU 硬件已在 main.c 中于 FreeRTOS 启动前初始化完成，
     * 避免在任务中调用 sh3001_init()/imu_init() 导致 delay_ms() 阻塞调度。
     * 传感器保持常开，不进入睡眠，无需唤醒操作。
     */
    g_sensor_state.initialized = 1;
    return 1;
}
```

**关键设计**：

- SH3001/IMU 硬件在 `main.c` 中于 FreeRTOS 启动前初始化
- 传感器保持常开，不进入睡眠
- 避免反复休眠/唤醒带来的 I2C 时序问题和首次数据不稳定

### 1.6 数据更新流程

```c
static void sensor_app_update(void)
{
    /* 1. 读取传感器数据 */
    g_sensor_state.temperature = sh3001_get_temperature();
    sh3001_get_imu_compdata(acc, gyro);

    /* 2. IMU 数据校准 */
    imu_data_calibration(&gyro[0], &gyro[1], &gyro[2],
                         &acc[0],  &acc[1],  &acc[2]);

    /* 3. Mahony AHRS 姿态解算 */
    g_sensor_state.angles = imu_get_eulerian_angles(gyro[0], gyro[1], gyro[2],
                                                     acc[0],  acc[1],  acc[2]);

    /* 4. 读取磁力计（约15ms，注意阻塞桌面任务） */
    st480mc_read_magdata(&mag[0], &mag[1], &mag[2]);

    /* 5. 设置数值（组件内部比较新旧值，仅变化时标记 dirty） */
    ui_data_row_group_setf(&s_temp_grp,     0, "%2.1f C", g_sensor_state.temperature);
    ui_data_row_group_setf(&s_attitude_grp, 0, "%6.1f", a->pitch);
    /* ... */

    /* 6. 批量绘制（仅 dirty 行触发 rcmd） */
    rcmd_begin();
    ui_data_row_group_draw(&s_temp_grp);
    ui_data_row_group_draw(&s_attitude_grp);
    /* ... */
    rcmd_flush();

    /* 7. 串口数据上报 */
    sensor_report_to_pc();
}
```

### 1.7 串口数据上报

```c
static void sensor_report_to_pc(void)
{
    if (!g_report_enabled)
    {
        return;
    }

    /* 每5次update发送一次，即500ms间隔 */
    g_report_counter++;
    if (g_report_counter < 5)
    {
        return;
    }
    g_report_counter = 0;

    ring_log_task("PITCH:%.2f,ROLL:%.2f,YAW:%.2f,AX:%d,AY:%d,AZ:%d,...",
                  a->pitch, a->roll, a->yaw,
                  acc[0], acc[1], acc[2],
                  gyro[0], gyro[1], gyro[2],
                  mag[0], mag[1], mag[2]);
}
```

## 2. 音乐播放器 (music_app)

### 2.1 功能概述

音乐播放器应用通过 USB Audio 设备模式，使用 ES8388 音频编解码器 + SAI 接口输出音频。支持音量控制、播放/暂停等功能。

实现位于 [`User/APP/music_player/music_app.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/music_player/music_app.c)。

### 2.2 异步初始化模式

音乐播放器采用**异步初始化模式**：先显示 UI，后台初始化 USB 音频硬件。

```c
static void music_app_enter(void)
{
    /* 第 1 步：立即显示 UI，让用户先看到界面 */
    rcmd_begin();
    titlebar_draw(STR_MUSIC_TITLE, &titlebar_style_default);
    music_ui_show();
    rcmd_flush();

    /* 第 2 步：后台初始化硬件（含 USB 枚举等待） */
    if (!g_music_app_state.hw_initialized)
    {
        if (music_player_init() == 0)
        {
            g_music_app_state.hw_initialized = 1;
        }
    }

    g_music_app_state.last_volume = music_player_get_volume();

    /* 第 3 步：更新 USB 连接状态 */
    rcmd_begin();
    music_ui_update_usb_state(g_device_state);
    rcmd_flush();
}
```

**异步初始化优势**：

- 用户无需等待硬件初始化完成即可看到界面
- USB 枚举可能需要数秒，异步避免阻塞
- 初始化失败时 UI 可以显示错误状态

### 2.3 状态管理

```c
static struct
{
    uint8_t initialized;
    uint8_t hw_initialized;     /* 硬件是否已完成初始化 */
    uint8_t last_volume;
} g_music_app_state;
```

### 2.4 周期性更新

```c
static void music_app_update(void)
{
    uint8_t current_volume;

    /* 执行音乐播放器业务逻辑 */
    music_player_task();

    /* 获取当前音量 */
    current_volume = music_player_get_volume();
    g_music_app_state.last_volume = current_volume;

    /* 更新 UI */
    rcmd_begin();
    music_ui_update_volume(current_volume);
    music_ui_update_usb_state(g_device_state);
    rcmd_flush();
}
```

## 3. 相机应用 (camera_app)

### 3.1 功能概述

相机应用通过 OV5640 摄像头模块和 DCMI 接口实现实时预览。采用分阶段初始化：先显示加载画面，再阻塞初始化硬件。

实现位于 [`User/APP/camera_app/camera_app.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/camera_app/camera_app.c)。

### 3.2 初始化阶段

```c
#define CAMERA_INIT_PHASE_INIT    0   /* 正在初始化 */
#define CAMERA_INIT_PHASE_OK      1   /* 初始化成功，正常运行 */
#define CAMERA_INIT_PHASE_FAIL    2   /* 初始化失败 */
```

### 3.3 分阶段初始化流程

```c
static void camera_app_enter(void)
{
    /* 防止触摸中断初始化流程 */
    g_camera_state.is_visible = 0;
    g_camera_state.init_phase = CAMERA_INIT_PHASE_INIT;

    /* 先绘制初始化画面 */
    rcmd_begin();
    camera_ui_draw_init_screen();
    rcmd_flush();

    /* 阻塞执行硬件初始化 */
    if (ov5640_init() == 0)
    {
        /* OV5640 初始化成功 */
        dcmi_init();

        /* 配置 OV5640 参数 */
        ov5640_rgb565_mode();
        ov5640_light_mode(0);
        ov5640_color_saturation(3);
        ov5640_brightness(4);
        ov5640_contrast(3);
        ov5640_sharpness(33);
        ov5640_outsize_set(16, 4, lcddev.width, camera_ui_get_preview_height());

        /* 启动预览 */
        camera_ui_start();
        camera_ui_setup_dcmi_dma();
        dcmi_start();

        g_camera_state.is_visible = 1;
        g_camera_state.init_phase = CAMERA_INIT_PHASE_OK;
    }
    else
    {
        /* OV5640 初始化失败 */
        g_camera_state.is_visible = 1;
        g_camera_state.init_phase = CAMERA_INIT_PHASE_FAIL;

        rcmd_begin();
        camera_ui_draw_error_screen();
        rcmd_flush();
    }
}
```

**设计要点**：

1. **先显示加载画面**：避免用户看到白屏
2. **is_visible = 0**：初始化期间禁止触摸处理
3. **阻塞初始化**：相机硬件需要严格时序，必须阻塞
4. **错误处理**：初始化失败时显示错误画面

### 3.4 退出处理

```c
static void camera_app_exit(void)
{
    g_camera_state.is_visible = 0;

    if (g_camera_state.init_phase == CAMERA_INIT_PHASE_OK)
    {
        dcmi_stop();
        camera_ui_stop();
    }

    camera_app_deinit();
}
```

仅在初始化成功时才停止 DCMI，避免对未初始化的硬件操作。

## 4. 文件管理器 (file_manager)

### 4.1 功能概述

文件管理器基于 FATFS 文件系统，浏览 NORFlash/SD 卡文件，支持目录进入/返回、文件列表分页。

实现分布在：

- [`User/APP/file_manager/file_manager_app.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/file_manager/file_manager_app.c) — 应用包装层
- [`User/APP/file_manager/file_manager_ui.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/file_manager/file_manager_ui.c) — UI 层
- [`User/APP/file_manager/file_manager_logic.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/file_manager/file_manager_logic.c) — 业务逻辑层

### 4.2 状态结构

```c
typedef struct
{
    uint8_t     initialized;
    uint8_t     is_visible;
    fm_state_t  state;
} g_fm_app_state;
```

### 4.3 触摸处理

```c
static uint8_t file_manager_app_handle_touch(uint16_t x, uint16_t y)
{
    uint8_t idx;
    const fm_file_item_t *item;

    if (!g_fm_app_state.is_visible)
    {
        return 0xFF;
    }

    /* 返回按钮 */
    if (titlebar_is_back_touched(x, y))
    {
        return 0;
    }

    /* 文件列表点击 */
    idx = fm_ui_hit_test(x, y, &g_fm_app_state.state);
    if (idx == 0xFF)
    {
        return 0xFF;
    }

    item = &g_fm_app_state.state.items[idx];

    /* 选中高亮 */
    g_fm_app_state.state.sel_index = idx;
    rcmd_begin();
    fm_ui_draw_list(&g_fm_app_state.state);
    rcmd_flush();

    /* 处理进入/返回 */
    if (item->is_dir)
    {
        if (item->is_dot)
        {
            /* ".." 返回上级 */
            fm_logic_go_up(&g_fm_app_state.state);
        }
        else
        {
            /* 进入子目录 */
            fm_logic_enter_dir(&g_fm_app_state.state, item->name);
        }

        /* 标记需要刷新，update 中处理 */
        g_fm_app_state.state.need_refresh = 1;
    }
    else
    {
        /* 点击文件：仅日志记录 */
        ring_log_task("[FM] select file: %s, size=%lu", item->name, item->size);
    }

    return 0xFF;
}
```

**设计要点**：

- 目录操作标记 `need_refresh = 1`，在 `update()` 中统一刷新
- 文件点击仅记录日志，后续可扩展为打开/预览
- 选中高亮立即反馈，目录切换延迟到 `update()`

## 5. 应用开发规范

### 5.1 目录结构

新应用在 `User/APP/` 下创建独立文件夹：

```
User/APP/
├── app_interface.h          # 系统接口（只读）
├── app_registry.c           # 注册新应用（修改）
├── music_player/            # 示例
│   ├── music_app.c/h        # 包装层（必须）
│   ├── music_player.c/h     # 业务逻辑（可选）
│   └── music_ui.c/h         # UI层（可选）
└── your_app/
    ├── your_app.c/h         # 包装层（必须）
```

| 文件 | 职责 | 必须 |
|------|------|------|
| `your_app.c` | 实现 `app_interface_t`，组装 UI 和业务逻辑 | ✅ |
| `your_app.h` | 暴露 `your_app_get_interface()` | ✅ |
| `your_ui.c` | UI 渲染（`rcmd_xxx` 绘制） | ❌ |
| `your_logic.c` | 业务逻辑、硬件控制 | ❌ |

```c
#include "APP/hello_app/hello_app.h"
#include "RENDER/render_cmd.h"
#include "app_strings.h"

static struct { uint8_t initialized; uint8_t is_visible; } g_hello_state;

static uint8_t hello_app_init(void)
{
    if (g_hello_state.initialized) return 1;
    g_hello_state.initialized = 1;
    return 1;
}

static void hello_app_deinit(void)
{
    g_hello_state.initialized = 0;
    g_hello_state.is_visible = 0;
}

static void hello_app_enter(void)
{
    g_hello_state.is_visible = 1;
    rcmd_begin();
    rcmd_fill(0, DESKTOP_STATUSBAR_HEIGHT, lcddev.width - 1,
              lcddev.height - DESKTOP_TASKBAR_HEIGHT - 1, 0x1082);
    rcmd_text(50, 100, 200, 24, 16, "Hello, FreeOS!", WHITE);
    rcmd_flush();
}

static void hello_app_exit(void)
{
    g_hello_state.is_visible = 0;
    hello_app_deinit();
}

static void hello_app_update(void)
{
    if (!g_hello_state.is_visible) return;
}

static uint8_t hello_app_handle_touch(uint16_t x, uint16_t y)
{
    if (!g_hello_state.is_visible) return 0xFF;
    if (x >= 10 && x <= 70 && y >= DESKTOP_STATUSBAR_HEIGHT + 5 &&
        y <= DESKTOP_STATUSBAR_HEIGHT + 35)
        return 0;
    return 0xFF;
}

static const app_interface_t s_hello_app_interface = {
    .name = STR_APP_NAME_HELLO,
    .icon_type = ICON_TYPE_APP,
    .init = hello_app_init,
    .deinit = hello_app_deinit,
    .enter = hello_app_enter,
    .exit = hello_app_exit,
    .update = hello_app_update,
    .handle_touch = hello_app_handle_touch,
};

const app_interface_t *hello_app_get_interface(void) { return &s_hello_app_interface; }
```

### 5.3 接口说明

| 函数 | 调用时机 | 职责 |
|------|----------|------|
| `init` | 首次启动 | 分配资源、检查 `initialized` 避免重复 |
| `deinit` | 退出/关机 | 释放资源、清 `initialized` |
| `enter` | 切换到前台 | 绘制界面、设置 `is_visible = 1` |
| `exit` | 返回桌面 | 停硬件、设 `is_visible = 0`、调用 `deinit` |
| `update` | 每 100ms | 更新数据、刷新 UI，执行时间 **< 50ms** |
| `handle_touch` | 按下时 | 检测返回按钮返回 `0`，其余返回 `0xFF` |

### 5.4 enter() 的两种模式

| 模式 | 适用场景 | 示例 |
|------|----------|------|
| **即时显示** | 硬件已初始化，直接绘制完整界面 | sensor_app |
| **异步初始化** | 硬件初始化耗时，先绘加载画面 | camera_app、music_app |

### 5.10 FreeRTOS 注意事项

#### 调度器启动前初始化

含 `delay_ms()` 的硬件初始化必须放在 `main.c` 的 `freertos_demo()` 之前：

```c
ov5640_init();      /* ~300ms delay */
sh3001_init();      /* ~270ms delay */
freertos_demo();    /* 启动调度器 */
```

#### 中断优先级

- 使用 FreeRTOS API 的中断：优先级 **<= 5**
- 不使用 FreeRTOS API 的中断：优先级可以 **> 5**

#### update() 限制

- 避免在 `update()` 中使用 `delay_ms()`、`vTaskDelay()`、轮询等待
- 执行时间过长会阻塞桌面任务，影响状态栏刷新

### 5.11 中断与并发

- ISR 与任务共享的变量加 `volatile`
- ISR 中只做数据搬运，避免复杂逻辑
- ISR 中调用 FreeRTOS API 必须使用 `FromISR` 版本
- 覆写全局回调指针（如 `dcmi_rx_callback`）应在硬件启动前完成，退出时置 NULL

### 5.12 日志规范

| 上下文 | 函数 | 说明 |
|--------|------|------|
| 任务 | `ring_log_task("[app] msg")` | 可阻塞等待缓冲 |
| 中断 | `ring_log_isr("[app] msg")` | 无阻塞 |

- 日志前缀用 `[appname]` 便于过滤
- 避免在 `update()` 中高频打印

### 5.13 注意事项

1. 不要修改 `app_interface.h`
2. 不要直接 include `desktop_app.h`，通过 `app_interface.h` 交互
3. 避免使用无 `static` 修饰的全局变量
4. `update()` 执行时间应 `< 50ms`
5. 避免在中断中调用 `delay_ms()`、`printf()`、`malloc/free`
6. DMA 缓冲区使用 `mymalloc(SRAMIN, ...)` 分配，确保在 AXI SRAM

### 5.14 调试指南

| 问题 | 诊断方法 |
|------|----------|
| 串口日志 | USART1 (115200)，使用 `ring_log_task()` / `ring_log_isr()` |
| 栈溢出 | 触发 `sys_panic()`，LED 红灯快闪 |
| 断言失败 | 串口输出文件名/行号/任务名，任务上下文仅终止当前任务 |
| 白屏/黑屏 | 检查 `rcmd_begin/rcmd_flush` 是否成对 |
| 触摸无响应 | 检查 `handle_touch()` 返回值（`0` 才触发返回） |
| 内存分配失败 | 检查 `mymalloc()` 返回值，释放已分配资源 |
| init() 返回 0 | 桌面不会进入应用，检查串口日志 |

## 6. 本章小结

本章详细介绍了 FreeOS Desktop 的四个现有应用和开发规范。关键要点：

1. **传感器应用**：使用 `ui_data_row_group` 组件展示多组传感器数据，支持串口上报
2. **音乐播放器**：采用异步初始化模式，先显示 UI 再后台初始化硬件
3. **相机应用**：分阶段初始化，先显示加载画面，再阻塞初始化硬件
4. **文件管理器**：三层架构（包装层 + UI 层 + 逻辑层），目录操作延迟刷新
5. **开发规范**：统一的应用接口、状态结构体模板、UI 绘制规范
6. **注意事项**：调度器启动前初始化、中断优先级、update() 执行时间限制
