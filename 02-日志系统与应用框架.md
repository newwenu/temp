# FreeOS Desktop 技术手册 —— 第二部分：日志系统、应用框架与桌面架构

## 1. ISR 安全日志系统

### 1.1 概述

日志系统是嵌入式系统调试和诊断的核心基础设施。FreeOS Desktop 实现了一个 **ISR 安全的环形缓冲区日志系统**，支持任务级和中断级双模式写入，并集成了 LED 运行状态指示功能。

日志系统实现位于 [`User/ring_log.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/ring_log.c) 和 [`User/ring_log.h`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/ring_log.h)。

### 1.2 设计特点

| 特性 | 说明 |
|------|------|
| **ISR 安全** | 支持中断上下文和任务上下文两种写入模式 |
| **环形缓冲** | 2KB 固定大小环形缓冲区，避免内存动态分配 |
| **非阻塞丢弃** | 缓冲区满时丢弃新数据，不阻塞调用者 |
| **LED 指示** | 绿灯心跳闪烁（500ms 周期），断言/栈溢出时红灯快闪 |
| **自动复位** | 系统 panic 后约 5 秒自动复位 |

### 1.3 核心数据结构

```c
#define RING_LOG_SIZE       2048    /* 环形缓冲区大小 */
#define RING_LOG_TASK_PRIO  1       /* 日志任务优先级（低优先级） */
#define RING_LOG_TASK_STK   256     /* 日志任务栈大小 */
#define LED_HEARTBEAT_MS    500     /* LED心跳周期 */

static volatile uint16_t g_ring_head = 0;   /* 写指针 */
static volatile uint16_t g_ring_tail = 0;   /* 读指针 */
static char g_ring_buf[RING_LOG_SIZE];       /* 环形缓冲区 */
static volatile uint8_t g_ring_log_ready = 0; /* 初始化完成标志 */
```

### 1.4 API 接口

```c
/* 初始化日志后台任务和LED指示 */
void ring_log_init(void);

/* 中断安全日志写入（可在ISR中调用） */
void ring_log_isr(const char *fmt, ...);

/* 任务级日志写入 */
void ring_log_task(const char *fmt, ...);

/* 强制刷新缓冲区 */

/* 系统致命错误指示（断言/栈溢出时调用，会关中断并闪灯） */
void sys_panic(void);

/* 系统错误指示（单次调用，点亮红灯） */
void sys_error_led(void);
```

### 1.5 任务级日志写入

```c
void ring_log_task(const char *fmt, ...)
{
    char buf[128];
    va_list args;

    if (!g_ring_log_ready)
    {
        return;     /* 未初始化完成,直接丢弃 */
    }

    va_start(args, fmt);
    vsnprintf(buf, sizeof(buf), fmt, args);
    va_end(args);

    taskENTER_CRITICAL();
    ring_log_write(buf);
    taskEXIT_CRITICAL();
}
```

**特点**：

- 使用 `taskENTER_CRITICAL()` / `taskEXIT_CRITICAL()` 保护环形缓冲区操作
- 128 字节临时缓冲区用于格式化字符串
- 未初始化时静默丢弃，防止在初始化前被调用导致异常

### 1.6 中断级日志写入

```c
void ring_log_isr(const char *fmt, ...)
{
    char buf[128];
    va_list args;
    UBaseType_t uxSavedInterruptStatus;

    if (!g_ring_log_ready)
    {
        return;     /* 未初始化完成,直接丢弃,防止在初始化前被中断调用 */
    }

    va_start(args, fmt);
    vsnprintf(buf, sizeof(buf), fmt, args);
    va_end(args);

    uxSavedInterruptStatus = taskENTER_CRITICAL_FROM_ISR();
    ring_log_write(buf);
    taskEXIT_CRITICAL_FROM_ISR(uxSavedInterruptStatus);
}
```

**特点**：

- 使用 `taskENTER_CRITICAL_FROM_ISR()` / `taskEXIT_CRITICAL_FROM_ISR()` 保护
- 保存并恢复中断状态，支持中断嵌套
- 与任务级日志使用相同的环形缓冲区

### 1.7 环形缓冲区写入

```c
static void ring_log_putchar(char c)
{
    uint16_t next = (g_ring_head + 1) % RING_LOG_SIZE;
    if (next != g_ring_tail)
    {
        g_ring_buf[g_ring_head] = c;
        g_ring_head = next;
    }
}

static void ring_log_write(const char *str)
{
    while (*str)
    {
        ring_log_putchar(*str++);
    }
    ring_log_putchar('\r');
    ring_log_putchar('\n');
}
```

**缓冲区满处理策略**：

- 当 `next == g_ring_tail` 时，表示缓冲区已满
- 新数据被静默丢弃，不阻塞调用者
- 这种设计确保在中断上下文中不会阻塞

### 1.8 后台日志任务

```c
static void ring_log_task_entry(void *pvParameters)
{
    TickType_t last_blink = xTaskGetTickCount();
    (void)pvParameters;

    while (1)
    {
        /* 刷新日志缓冲区 */
        taskENTER_CRITICAL();
        while (g_ring_tail != g_ring_head)
        {
            char c = g_ring_buf[g_ring_tail];
            g_ring_tail = (g_ring_tail + 1) % RING_LOG_SIZE;
            taskEXIT_CRITICAL();

            putchar(c);     /* 输出到串口 */

            taskENTER_CRITICAL();
        }
        taskEXIT_CRITICAL();

        /* LED心跳指示 */
        if ((xTaskGetTickCount() - last_blink) >= pdMS_TO_TICKS(LED_HEARTBEAT_MS))
        {
            LED1_TOGGLE();  /* 绿灯心跳 */
            last_blink = xTaskGetTickCount();
        }

        vTaskDelay(pdMS_TO_TICKS(10));
    }
}
```

**任务职责**：

1. **日志刷新**：周期性将环形缓冲区中的数据输出到串口
2. **LED 心跳**：每 500ms 翻转一次绿灯，指示系统正常运行
3. **低优先级**：优先级为 1，确保不影响其他业务任务

### 1.9 系统错误处理

#### sys_panic() — 致命错误处理

```c
void sys_panic(void)
{
    taskDISABLE_INTERRUPTS();
    LED0(0);  /* 红灯亮 (低电平点亮) */
    LED1(1);  /* 绿灯灭 (高电平熄灭) */

    /* 慢闪约 5 秒后自动复位，频率约 1Hz，方便人眼观察 */
    for (volatile int cnt = 0; cnt < 10; cnt++)
    {
        LED0_TOGGLE();
        for (volatile int i = 0; i < 20000000; i++);
    }

    NVIC_SystemReset();  /* 执行系统复位 */
}
```

**处理流程**：

1. 关闭所有中断
2. 点亮红灯，熄灭绿灯
3. 红灯以约 1Hz 频率慢闪 10 次（约 5 秒）
4. 执行系统复位

**触发条件**：

- 断言失败（中断上下文）
- 栈溢出检测
- `vAssertCalled` 中判断为中断上下文时

#### sys_error_led() — 错误指示

```c
void sys_error_led(void)
{
    LED0(0);  /* 红灯亮 (低电平点亮) */
}
```

单次调用点亮红灯，用于指示错误状态但不触发复位。

### 1.10 初始化

```c
void ring_log_init(void)
{
    g_ring_head = 0;
    g_ring_tail = 0;
    memset(g_ring_buf, 0, sizeof(g_ring_buf));

    g_log_task_handle = xTaskCreateStatic(
        ring_log_task_entry,
        "log_task",
        RING_LOG_TASK_STK,
        NULL,
        RING_LOG_TASK_PRIO,
        g_log_task_stack,
        &g_log_task_tcb
    );

    g_ring_log_ready = 1;
}
```

**注意**：`g_ring_log_ready` 在任务创建后才置 1，确保任务已经就绪可以处理日志。

## 2. 应用框架

### 2.1 设计目标

应用框架的设计目标是实现**桌面与具体应用的解耦**。所有应用必须实现统一的接口结构体，桌面通过该接口管理应用的生命周期，无需了解应用内部实现细节。

### 2.2 应用接口定义

应用接口定义位于 [`User/APP/app_interface.h`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/app_interface.h)。

```c
typedef struct
{
    const char *name;                           /* 应用名称 */
    uint8_t icon_type;                          /* 桌面图标类型，使用 ICON_TYPE_XXX */

    uint8_t (*init)(void);                      /* 初始化，返回 1=成功 */
    void (*deinit)(void);                       /* 反初始化 */
    void (*enter)(void);                        /* 进入前台 */
    void (*exit)(void);                         /* 退出前台 */
    void (*update)(void);                       /* 每 100ms 调用 */
    uint8_t (*handle_touch)(uint16_t x, uint16_t y); /* 触摸处理，0=返回桌面 */
} app_interface_t;
```

### 2.3 接口成员说明

| 成员 | 类型 | 说明 |
|------|------|------|
| `name` | `const char *` | 应用名称，用于桌面显示和查找 |
| `icon_type` | `uint8_t` | 桌面图标类型，使用 `ICON_TYPE_XXX` 枚举 |
| `init` | `uint8_t (*)(void)` | 应用初始化，返回 1 表示成功，0 表示失败 |
| `deinit` | `void (*)(void)` | 应用反初始化，释放资源 |
| `enter` | `void (*)(void)` | 应用进入前台，执行首次绘制 |
| `exit` | `void (*)(void)` | 应用退出前台，清理状态 |
| `update` | `void (*)(void)` | 周期性更新（每 100ms），用于动态数据刷新 |
| `handle_touch` | `uint8_t (*)(uint16_t, uint16_t)` | 触摸事件处理，返回 0 表示返回桌面 |

### 2.4 应用生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                     应用生命周期状态机                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [未注册] ──register──► [已注册] ──init──► [已初始化]      │
│                                                             │
│   [已初始化] ──enter──► [运行中] ──update──► [运行中]       │
│        ▲                    │                               │
│        │                   exit                              │
│        │                    │                               │
│        └──────────── [已退出] ◄──exit── [运行中]             │
│                                                             │
│   [已初始化/已退出] ──deinit──► [未初始化]                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**生命周期说明**：

1. **注册**：应用在 `app_registry.c` 中通过 `app_manager_register()` 注册
2. **初始化**：`init()` 分配资源、初始化硬件，返回 1 表示成功
3. **进入**：`enter()` 执行首次 UI 绘制，应用进入前台
4. **更新**：`update()` 每 100ms 被调用，用于刷新动态数据
5. **触摸**：`handle_touch()` 处理触摸事件，返回 0 触发返回桌面
6. **退出**：`exit()` 清理前台状态，返回桌面
7. **反初始化**：`deinit()` 释放所有资源

### 2.5 应用管理器

应用管理器实现位于 [`User/APP/app_manager.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/APP/app_manager.c)，提供线程安全的应用注册、查询和生命周期管理。

#### 2.5.1 线程安全设计

```c
static SemaphoreHandle_t g_app_mutex = NULL;
static StaticSemaphore_t g_app_mutex_buffer;

static void app_manager_lock(void)
{
    xSemaphoreTake(g_app_mutex, portMAX_DELAY);
}

static void app_manager_unlock(void)
{
    xSemaphoreGive(g_app_mutex);
}
```

使用 FreeRTOS 互斥量（静态分配）保护共享状态，确保多任务安全访问。

#### 2.5.2 应用注册

```c
void app_manager_register(const app_interface_t *app)
{
    if (app == NULL || g_app_count >= APP_MAX_COUNT)
    {
        return;
    }

    app_manager_lock();
    g_app_table[g_app_count++] = app;
    app_manager_unlock();
}
```

- 最大应用数：`APP_MAX_COUNT = 8`
- 注册时检查空指针和数量上限

#### 2.5.3 应用查询

```c
uint8_t app_manager_get_count(void);
const app_interface_t *app_manager_get_by_index(uint8_t index);
const app_interface_t *app_manager_get_by_name(const char *name);
const app_interface_t *app_manager_get_current(void);
```

所有查询操作均在互斥量保护下进行。

#### 2.5.4 应用进入

```c
uint8_t app_manager_enter_app(const app_interface_t *app)
{
    uint8_t result = 0;
    uint8_t init_ok = 0;

    if (app == NULL)
    {
        return 0;
    }

    app_manager_lock();

    if (app == g_current_app)
    {
        app_manager_unlock();
        return 1;
    }

    if (g_current_app != NULL)
    {
        g_current_app->exit();
        g_current_app = NULL;
    }

    app_manager_unlock();

    /* 在锁外调用init，避免长时间阻塞其他任务访问app_manager */
    init_ok = app->init();

    if (init_ok)
    {
        app_manager_lock();
        app->enter();
        g_current_app = app;
        result = 1;
        app_manager_unlock();
    }

    return result;
}
```

**设计要点**：

1. **避免重复进入**：如果目标应用已是当前应用，直接返回成功
2. **先退出当前应用**：调用当前应用的 `exit()` 后再进入新应用
3. **锁外初始化**：`init()` 在互斥量外调用，避免长时间阻塞其他任务
4. **初始化失败保护**：如果 `init()` 返回 0，不执行 `enter()`

#### 2.5.5 应用退出

```c
void app_manager_exit_app(void)
{
    app_manager_lock();

    if (g_current_app != NULL)
    {
        g_current_app->exit();
        g_current_app = NULL;
    }

    app_manager_unlock();
}
```

## 3. 桌面双任务架构

### 3.1 架构概述

桌面采用**双任务架构**，将 UI 更新和触摸处理分离到两个独立任务中：

| 任务 | 优先级 | 周期 | 职责 |
|------|--------|------|------|
| `desktop_task` | 2 | 100ms | 调用当前应用的 `update()`，或桌面 `desktop_update()` |
| `touch_task` | 3 | 10ms | 触摸扫描、应用切换、触摸事件分发 |

### 3.2 任务配置

```c
#define DESKTOP_TASK_PRIO       2
#define DESKTOP_TASK_STK_SIZE   1024

#define TOUCH_TASK_PRIO         3
#define TOUCH_TASK_TASK_STK_SIZE 1024
```

两个任务均使用静态内存分配。

### 3.3 桌面任务

```c
void desktop_task(void *pvParameters)
{
    TickType_t xLastWakeTime = xTaskGetTickCount();
    const TickType_t xFrequency = pdMS_TO_TICKS(100);

    while (1)
    {
        const app_interface_t *current = app_manager_get_current();

        if (current != NULL)
        {
            current->update();
        }
        else
        {
            desktop_update();
        }

        vTaskDelayUntil(&xLastWakeTime, xFrequency);
    }
}
```

**设计要点**：

- 使用 `vTaskDelayUntil()` 实现固定周期调度（100ms）
- 如果当前有应用运行，调用应用的 `update()`
- 如果当前无应用运行（桌面状态），调用 `desktop_update()`
- 优先级为 2，低于触摸任务

### 3.4 触摸任务

```c
void touch_task(void *pvParameters)
{
    uint8_t touch_flag = 0;
    TickType_t xLastWakeTime = xTaskGetTickCount();
    const TickType_t xFrequency = pdMS_TO_TICKS(10);

    while (1)
    {
        tp_dev.scan(0);

        if (tp_dev.sta & TP_PRES_DOWN)
        {
            uint16_t x = tp_dev.x[0];
            uint16_t y = tp_dev.y[0];

            if (!touch_flag)
            {
                touch_flag = 1;

                const app_interface_t *current = app_manager_get_current();

                if (current != NULL)
                {
                    /* 应用运行中 */
                    uint16_t nav_y = lcddev.height - DESKTOP_TASKBAR_HEIGHT;
                    if (y >= nav_y)
                    {
                        /* 底部导航栏区域，由桌面处理 */
                        desktop_handle_touch(x, y);
                    }
                    else
                    {
                        /* 应用内容区域，分发给应用 */
                        uint8_t btn_id = current->handle_touch(x, y);
                        if (btn_id == 0)
                        {
                            desktop_return_to_home();
                        }
                    }
                }
                else
                {
                    /* 桌面状态 */
                    uint8_t icon_index = desktop_get_icon_at(x, y);

                    if (icon_index != 0xFF)
                    {
                        /* 点击了应用图标 */
                        desktop_select_icon(icon_index);

                        const app_interface_t *target = 
                            app_manager_get_by_name(desktop_icons[icon_index].name);

                        if (target != NULL)
                        {
                            vTaskDelay(pdMS_TO_TICKS(200));  /* 视觉反馈延迟 */
                            app_manager_enter_app(target);
                        }
                    }
                    else
                    {
                        /* 点击了空白区域 */
                        desktop_handle_touch(x, y);
                    }
                }
            }
        }
        else
        {
            touch_flag = 0;  /* 触摸释放 */
        }

        vTaskDelayUntil(&xLastWakeTime, xFrequency);
    }
}
```

**触摸处理逻辑**：

1. **触摸扫描**：每 10ms 调用 `tp_dev.scan(0)` 读取触摸状态
2. **按下检测**：使用 `touch_flag` 实现边沿检测，只在首次按下时处理
3. **应用运行中**：
   - 底部导航栏区域（`y >= nav_y`）：由桌面处理返回/主页事件
   - 应用内容区域：调用应用的 `handle_touch()`，返回 0 触发返回桌面
4. **桌面状态**：
   - 点击应用图标：高亮图标，延迟 200ms 后进入应用
   - 点击空白区域：由桌面处理
5. **触摸释放**：清除 `touch_flag`，准备下一次检测

### 3.5 桌面初始化


5. 根据注册的应用自动添加桌面图标

### 3.7 导航事件处理

```c
static void desktop_app_nav_event_handler(nav_event_t event)
{
    switch (event)
    {
        case NAV_EVENT_BACK:
        case NAV_EVENT_HOME:
            if (app_manager_get_current() != NULL)
            {
                desktop_return_to_home();
            }
            break;

        case NAV_EVENT_RECENT:
            break;

        default:
            break;
    }
}
```

## 4. 本章小结

本章介绍了 FreeOS Desktop 的日志系统、应用框架和桌面双任务架构。关键要点：

1. **ISR 安全日志**：2KB 环形缓冲区，支持任务级和中断级双模式写入，带 LED 心跳指示
2. **统一应用接口**：`app_interface_t` 定义了应用的标准生命周期，实现桌面与应用的解耦
3. **线程安全**：应用管理器使用互斥量保护共享状态，支持多任务安全访问
4. **双任务架构**：桌面任务（100ms 周期）负责 UI 更新，触摸任务（10ms 周期）负责输入处理
5. **触摸分发**：根据坐标区域决定由桌面还是当前应用处理触摸事件
6. **静态分配**：所有任务和同步原语均使用静态内存分配
