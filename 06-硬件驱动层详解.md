## 1. LCD 显示驱动

### 1.1 架构概述

FreeOS Desktop 支持两种 LCD 接口模式：

| 接口类型 | 适用场景 | 分辨率支持 | 驱动文件 |
|---------|---------|-----------|---------|
| MCU 接口 (8080并行) | 中小尺寸屏 (2.8~4.3寸) | 最高 480x320 | `lcd.c/h` |
| LTDC RGB 接口 | 大尺寸屏 (7寸及以上) | 最高 1280x800 | `ltdc.c/h` |

系统通过 `lcddev` 全局结构体统一管理 LCD 参数，上层应用调用统一的 `lcd_xxx()`  API，无需区分底层接口类型。

### 1.2 MCU 接口驱动 (`lcd.c`)

#### 1.2.1 核心结构体

```c
typedef struct
{
    uint16_t width;       /* LCD 宽度 */
    uint16_t height;      /* LCD 高度 */
    uint16_t id;          /* LCD ID */
    uint8_t  dir;         /* 显示方向: 0-横屏, 1-竖屏 */
    uint16_t wramcmd;     /* 开始写 GRAM 指令 */
    uint16_t setxcmd;     /* 设置 X 坐标指令 */
    uint16_t setycmd;     /* 设置 Y 坐标指令 */
} _lcd_dev;
```

#### 1.2.2 关键操作函数

```c
void lcd_wr_data(volatile uint16_t data);        /* 写数据 */
void lcd_wr_regno(volatile uint16_t regno);      /* 写寄存器地址 */
void lcd_write_reg(uint16_t regno, uint16_t data); /* 写寄存器 */
static uint16_t lcd_rd_data(void);                /* 读数据 */
void lcd_write_ram_prepare(void);                 /* 准备写 GRAM */
```

**线程安全**: 所有底层寄存器操作均使用 `taskENTER_CRITICAL()` / `taskEXIT_CRITICAL()` 保护，确保在 FreeRTOS 多任务环境下的原子性。

#### 1.2.3 支持的驱动 IC

| IC 型号 | 分辨率 | 特点 |
|--------|--------|------|
| ILI9341 | 240x320 | 经典 2.8寸屏驱动 |
| ST7789 | 240x320 | 高性价比 |
| NT35310 | 480x320 | 3.5寸高分辨率 |
| NT35510 | 480x800 | 4.3寸 WVGA |
| SSD1963 | 800x480 | 支持大分辨率 |
| ST7796 | 480x320 | 3.5寸替代方案 |
| ILI9806 | 480x800 | 4.3寸替代方案 |

#### 1.2.4 扫描方向配置

```c

#define L2R_D2U    1    /* 从左到右,从下到上 */
```

DMA2D 可在不占用 CPU 的情况下完成：
- 矩形区域填充
- 颜色格式转换
- 图像拷贝/混合

## 2. 触摸屏驱动

### 2.1 架构概述

触摸屏驱动采用抽象层设计，支持电阻屏和电容屏两种类型，通过统一的 `_m_tp_dev` 结构体对外提供服务。

### 2.2 核心结构体

```c
typedef struct
{
    uint8_t (*init)(void);           /* 初始化触摸屏控制器 */
    uint8_t (*scan)(uint8_t);        /* 扫描: 0-屏幕扫描; 1-物理坐标 */
    void (*adjust)(void);            /* 触摸屏校准 */
    uint16_t x[CT_MAX_TOUCH];        /* 当前坐标 */
    uint16_t y[CT_MAX_TOUCH];        /* 电容屏最多10点,电阻屏用 x[0],y[0] */
    uint16_t sta;                    /* 触摸状态
                                      * b15: 按下1/松开0
                                      * b14: 0-无按键; 1-有按键按下
                                      * b9~b0: 电容屏按下的点数 */
    /* 5点校准参数 (仅电阻屏) */
    float xfac;
    float yfac;
    short xc;
    short yc;
    uint8_t touchtype;               /* b0: 0-电阻屏; 1-电容屏
                                      * b7: 0-未校准; 1-已校准 */
} _m_tp_dev;
```

#### 2.3.1 工作原理

### 2.4 电容屏驱动

#### 2.4.1 FT5206 驱动

FT5206 支持 5 点电容触摸，通过 I2C 通信：

```c
uint8_t ft5206_scan(uint8_t mode)
{
    static uint8_t t = 0;
    t++;
    
    /* 每 10 次查询才读取 1 次,降低 CPU 占用 */
    if ((t % 10) == 0 || t < 10)
    {
        ft5206_rd_reg(FT5206_REG_NUM_FINGER, &sta, 1);
        
        if ((sta & 0XF) && ((sta & 0XF) < 6))
        {
            /* 转换触摸点数为状态位 */
            temp = 0XFFFF << (sta & 0XF);
            tp_dev.sta = (~temp) | TP_PRES_DOWN | TP_CATH_PRES;
            
            /* 读取各点坐标 */
            for (i = 0; i < 5; i++)
            {
                ft5206_rd_reg(FT5206_TPX_TBL[i], buf, 4);
                tp_dev.x[i] = ((uint16_t)(buf[0] & 0X0F) << 8) + buf[1];
                tp_dev.y[i] = ((uint16_t)(buf[2] & 0X0F) << 8) + buf[3];
            }
        }
    }
}
```

**性能优化**: 使用静态变量 `t` 控制查询频率，每 10 次 `CTP_Scan` 调用才执行一次 I2C 读取，大幅降低 CPU 占用率。

#### 2.4.2 GT9XXX 系列驱动

GT9XXX 支持多点触摸（5点或10点），自动识别芯片型号：

```c
uint8_t gt9xxx_init(void)
{
    /* 复位序列 */
    GT9XXX_RST(0);
    delay_ms(10);
    GT9XXX_RST(1);
    delay_ms(100);
    
    /* 读取产品 ID */
    gt9xxx_rd_reg(GT9XXX_PRODUCT_ID_REG, temp, 4);
    
    /* 自动识别型号 */
    if (strcmp((char *)temp, "9147") == 0)  g_gt_tnum = 5;
    else if (strcmp((char *)temp, "917S") == 0) g_gt_tnum = 5;
    else if (strcmp((char *)temp, "968") == 0)  g_gt_tnum = 5;
    else if (strcmp((char *)temp, "1151") == 0) g_gt_tnum = 5;
    else if (strcmp((char *)temp, "9271") == 0) g_gt_tnum = 10;
    else return 1;  /* 不支持的 IC */
    
    /* 配置为轮询模式 */
    temp[0] = 0x01;
    gt9xxx_wr_reg(GT9XXX_INT_MODE_REG, temp, 1);
    return 0;
}
```

## 3. IMU 传感器驱动 (SH3001)

### 3.1 传感器概述

SH3001 是一款 6 轴惯性测量单元（IMU），集成：
- **三轴加速度计**: 测量线性加速度
- **三轴陀螺仪**: 测量角速度

通过 I2C 接口与 MCU 通信，用于传感器应用的姿态解算。

### 3.2 配置宏定义

#### 3.2.1 加速度计 ODR (输出数据率)

```c
#define SH3001_ODR_63HZ      (0x04)
#define SH3001_ODR_31HZ      (0x05)
#define SH3001_ODR_16HZ      (0x06)
#define SH3001_ODR_2000HZ    (0x08)

```

#### 3.2.2 加速度计量程

#### 3.2.3 陀螺仪量程

### 3.3 底层通信接口

```c

uint8_t sh3001_write_nbytes(uint8_t devaddr, uint8_t regaddr, uint8_t length, uint8_t *writebuf);
```

### 3.4 加速度计配置

```c
static void sh3001_acc_config(uint8_t acc_odr, uint8_t acc_range,
                               uint8_t acc_cutofffreq, uint8_t acc_filter_enble)
{
    /* 1. 使能加速度计数字滤波 */
    sh3001_read_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF0, 1, &regdata);
    regdata |= 0x01;
    sh3001_write_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF0, 1, &regdata);
    
    /* 2. 设置输出数据率 */
    sh3001_write_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF1, 1, &acc_odr);
    
    /* 3. 设置量程 */
    sh3001_write_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF2, 1, &acc_range);
    
    /* 4. 设置低通滤波器 */
    sh3001_read_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF3, 1, &regdata);
    regdata &= 0x17;
    regdata |= (acc_cutofffreq | acc_filter_enble);
    sh3001_write_nbytes(SH3001_ADDRESS, SH3001_ACC_CONF3, 1, &regdata);
}
```

### 3.5 数据读取与姿态解算

传感器应用使用 Mahony AHRS 算法进行姿态解算：

```
输入: 加速度 (ax, ay, az), 陀螺仪 (gx, gy, gz), 磁力计 (mx, my, mz)
输出: 俯仰角 (Pitch), 横滚角 (Roll), 航向角 (Yaw)
```

AHRS 算法通过融合三种传感器数据，补偿陀螺仪的零漂和加速度计的动态误差，提供稳定的姿态输出。

### 4.3 寄存器访问

ST480MC 采用特殊的 I2C 寄存器访问协议：

```c
uint8_t st480mc_write_register(uint8_t reg, uint16_t data)
{
    /* 发送写寄存器命令 */
    iic_send_byte(ST480MC_WRITE_REG);
    iic_send_byte(data >> 8);       /* 高字节 */
    iic_send_byte(data & 0XFF);     /* 低字节 */
    iic_send_byte(reg << 2);        /* 寄存器地址左移 2 位 */
}

uint16_t st480mc_read_register(uint8_t reg)
{
    /* 发送读寄存器命令 */
    iic_send_byte(ST480MC_READ_REG);
    iic_send_byte(reg << 2);
    /* 读取 3 字节: 状态 + 高字节 + 低字节 */
    if (buf[0] & 0X10) return 0XFFFF;  /* 错误标志 */
    return ((uint16_t)buf[1] << 8) | buf[2];
}
```

## 5. 摄像头驱动 (OV5640 + DCMI)

### 5.1 架构概述

摄像头模块采用 OV5640 传感器 + STM32 DCMI 接口 + DMA 传输的架构：

```
OV5640 传感器 --SCCB--> STM32H743 --DCMI--> DMA --> SDRAM/LCD
```

### 5.2 OV5640 驱动

#### 5.2.1 初始化流程

```c
uint8_t ov5640_init(void)
{
    /* 1. 初始化复位引脚 */
    /* 2. 通过 PCF8574 控制 PWDN 引脚上电 */
    /* 3. 硬件复位序列 */
    OV5640_RST(0); delay_ms(20);
    ov5640_pwdn_set(0); delay_ms(5);
    OV5640_RST(1); delay_ms(20);
    
    /* 4. 初始化 SCCB 接口 */
    sccb_init();
    
    /* 5. 读取芯片 ID 验证 */
    reg = ov5640_read_reg(OV5640_CHIPIDH) << 8;
    reg |= ov5640_read_reg(OV5640_CHIPIDL);
    if (reg != OV5640_ID) return 1;
    
    /* 6. 加载初始化寄存器表 (SXGA 1600x1200) */
    for (i = 0; i < sizeof(ov5640_init_reg_tbl) / 4; i++)
    {
        ov5640_write_reg(ov5640_init_reg_tbl[i][0], ov5640_init_reg_tbl[i][1]);
    }
    
    /* 7. 测试闪光灯 */
    ov5640_flash_ctrl(1); delay_ms(50);
    ov5640_flash_ctrl(0);
    return 0;
}
```

```c
void dcmi_dma_init(uint32_t mem0addr, uint32_t mem1addr,
                    uint16_t memsize, uint32_t memblen, uint32_t meminc)
{
    g_dma_dcmi_handle.Instance = DMA1_Stream1;
    g_dma_dcmi_handle.Init.Request = DMA_REQUEST_DCMI;
    g_dma_dcmi_handle.Init.Direction = DMA_PERIPH_TO_MEMORY;    /* 外设到内存 */
    g_dma_dcmi_handle.Init.PeriphInc = DMA_PINC_DISABLE;        /* 外设地址固定 */
    g_dma_dcmi_handle.Init.MemInc = meminc;                     /* 内存地址递增 */
    g_dma_dcmi_handle.Init.PeriphDataAlignment = DMA_PDATAALIGN_WORD;  /* 32bit */
    g_dma_dcmi_handle.Init.MemDataAlignment = memblen;          /* 8/16/32bit */
    g_dma_dcmi_handle.Init.Mode = DMA_CIRCULAR;                 /* 循环模式 */
    g_dma_dcmi_handle.Init.Priority = DMA_PRIORITY_HIGH;
    g_dma_dcmi_handle.Init.FIFOMode = DMA_FIFOMODE_ENABLE;      /* 使能 FIFO */
    HAL_DMA_Init(&g_dma_dcmi_handle);
}
```

## 6. 音频编解码器驱动 (ES8388)

### 6.1 芯片概述

ES8388 是一款低功耗音频编解码器，集成：
- **DAC**: 24bit 立体声数模转换，驱动耳机/扬声器
- **ADC**: 24bit 立体声模数转换，支持麦克风输入
- **接口**: I2S 音频数据接口 + I2C 控制接口

### 6.2 初始化流程

```c
uint8_t es8388_init(void)
{
    iic_init();                     /* 初始化 I2C */
    
    /* 软件复位 */
    es8388_write_reg(0, 0x80);
    es8388_write_reg(0, 0x00);
    sys_delay_ms(100);              /* RTOS 安全延时 */
    
    /* 电源管理配置 */
    es8388_write_reg(0x01, 0x58);
    es8388_write_reg(0x01, 0x50);
    es8388_write_reg(0x02, 0xF3);
    es8388_write_reg(0x02, 0xF0);
    
    /* ADC 配置 */
    es8388_write_reg(0x09, 0x88);   /* L/R PGA +24dB */
    es8388_write_reg(0x0C, 0x4C);   /* 16bit, 左数据=左 ADC */
    es8388_write_reg(0x0D, 0x02);   /* MCLK/采样率=256 */
    
    /* DAC 配置 */
    es8388_write_reg(0x17, 0x18);   /* 16bit */
    es8388_write_reg(0x18, 0x02);   /* MCLK/采样率=256 */
    es8388_write_reg(0x27, 0xB8);   /* L 音量 */
    es8388_write_reg(0x2A, 0xB8);   /* R 音量 */
    
    sys_delay_ms(100);
    return 0;
}
```

### 6.3 核心 API

```c

```

### 7.3 初始化序列

```c
void sdram_initialization_sequence(void)
{
    sdram_send_cmd(0, FMC_SDRAM_CMD_CLK_ENABLE, 1, 0);       /* 时钟使能 */
    delay_us(500);                                            /* 延时 500us */
    sdram_send_cmd(0, FMC_SDRAM_CMD_PALL, 1, 0);             /* 预充电所有 Bank */
    sdram_send_cmd(0, FMC_SDRAM_CMD_AUTOREFRESH_MODE, 8, 0); /* 自动刷新 8 次 */
    
    /* 加载模式寄存器 */
    temp = SDRAM_MODEREG_BURST_LENGTH_1 |
           SDRAM_MODEREG_BURST_TYPE_SEQUENTIAL |
           SDRAM_MODEREG_CAS_LATENCY_3 |
           SDRAM_MODEREG_OPERATING_MODE_STANDARD |
           SDRAM_MODEREG_WRITEBURST_MODE_SINGLE;
    sdram_send_cmd(0, FMC_SDRAM_CMD_LOAD_MODE, 1, temp);
}
```

## 8. NORFlash 驱动

### 8.1 硬件参数

| 参数 | 值 |
|------|-----|
| 芯片型号 | BY25Q256 / W25Q256 |
| 容量 | 32MB |
| 接口 | QSPI (四线 SPI) |
| 支持模式 | SPI / QPI |
| 最大时钟 | 104MHz (8 个 Dummy Clocks) |

### 8.2 QSPI 模式初始化

```c
void norflash_init(void)
{
    qspi_init();                        /* 初始化 QSPI 硬件 */
    norflash_qspi_disable();            /* 确保退出 QPI 模式 */
    norflash_qe_enable();               /* 使能 QE 位,启用 IO2/IO3 */
    g_norflash_type = norflash_read_id(); /* 读取芯片 ID */
    
    if (g_norflash_type == W25Q256 || g_norflash_type == BY25Q256)
    {
        /* 25Q256 需要 4 字节地址模式 */
        temp = norflash_read_sr(3);
        if ((temp & 0X01) == 0)
        {
            norflash_write_enable();
            qspi_send_cmd(FLASH_Enable4ByteAddr, 0, ...);
        }
        
        /* 设置读参数: 8 个 Dummy Clocks, 104MHz */
        norflash_write_enable();
        qspi_send_cmd(FLASH_SetReadParam, 0, ...);
        temp = 3 << 4;
        qspi_transmit(&temp, 1);
    }
}
```

### 8.3 状态寄存器

NORFlash 有 3 个状态寄存器：

**状态寄存器 1:**
| BIT | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|-----|---|---|---|---|---|---|---|---|
| | SPR | RV | TB | BP2 | BP1 | BP0 | WEL | BUSY |

**状态寄存器 2:**
| BIT | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|-----|---|---|---|---|---|---|---|---|
| | SUS | CMP | LB3 | LB2 | LB1 | (R) | QE | SRP1 |

**状态寄存器 3:**
| BIT | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|-----|---|---|---|---|---|---|---|---|
| | HOLD/RST | DRV1 | DRV0 | (R) | (R) | WPS | ADP | ADS |


## 9. MPU 内存保护配置

### 9.1 概述

STM32H743 的 MPU（Memory Protection Unit）用于配置各存储区域的访问权限、Cache 策略和缓冲属性。系统在启动时通过 `mpu_memory_protection()` 完成配置。

### 9.2 区域配置表

| 区域 | 基地址 | 大小 | 权限 | 可执行 | Cache | 缓冲 | 说明 |
|------|--------|------|------|--------|-------|------|------|
| 1 | 0x20000000 | 128KB | 全访问 | 是 | 是 | 是 | DTCM |
| 2 | 0x24000000 | 512KB | 全访问 | 是 | 是 | 是 | AXI SRAM |
| 3 | 0x30000000 | 512KB | 全访问 | 是 | 是 | 是 | SRAM1~3 |
| 4 | 0x38000000 | 64KB | 全访问 | 是 | 是 | 是 | SRAM4 |
| 5 | 0x60000000 | 64MB | 全访问 | 是 | 否 | 否 | FMC (LCD/SDRAM 控制) |
| 6 | 0xC0000000 | 32MB | 全访问 | 是 | 是 | 是 | SDRAM |
| 7 | 0x80000000 | 256MB | 全访问 | 否 | 否 | 否 | NAND FLASH |

### 9.3 配置函数

```c
uint8_t mpu_set_protection(uint32_t baseaddr, uint32_t size, uint32_t rnum,
                            uint8_t de, uint8_t ap, uint8_t sen, uint8_t cen, uint8_t ben);
```

**参数说明:**
- `baseaddr`: 区域基地址（32 字节对齐）
- `size`: 区域大小（参考 `CORTEX_MPU_Region_Size`）
- `rnum`: 区域编号（0~7）
- `de`: 禁止指令执行（0-允许; 1-禁止）
- `ap`: 访问权限（`MPU_REGION_FULL_ACCESS` 等）
- `sen`: 是否可共享
- `cen`: 是否可 Cache
- `ben`: 是否可缓冲

### 9.4 关键设计决策

1. **FMC 区域禁用 Cache**: LCD 寄存器通过 FMC 访问，必须禁用 Cache 避免数据不一致
2. **NAND FLASH 禁止执行**: 防止意外从 NAND 执行代码
3. **SDRAM 启用 Cache**: 提高图形数据访问效率

## 10. 硬件驱动使用总结

### 10.1 初始化顺序

系统在 `main.c` 中按以下顺序初始化硬件：

```
1. sys_cache_enable        /* 使能 Cache */
2. HAL_Init                /* HAL 库初始化 */
3. clock_init(400MHz)      /* 系统时钟 */
4. delay_init              /* 延时函数 */
5. usart_init(115200)      /* 串口调试 */
6. mpu_memory_protection   /* MPU 配置 */
7. led_init                /* LED 指示 */
8. sdram_init              /* 外部内存 */
9. lcd_init                /* 显示屏 */
10. startup_screen_show()  /* 开机画面 */
11. key_init               /* 按键 */
12. norflash_init          /* 外部存储 */
13. my_mem_init (6 pools)  /* 内存池 */
14. fonts_init             /* 字库 */
15. sh3001_init            /* IMU (RTOS 前!) */
16. st480mc_init           /* 磁力计 (RTOS 前!) */
17. freertos_demo()        /* 创建任务 */
18. vTaskStartScheduler    /* 启动调度器 */
```

**重要约束**: SH3001 和 ST480MC 的初始化必须在 FreeRTOS 调度器启动前完成，因为内部使用 `delay_ms()` 延时约 270ms，调度器启动后会导致任务死锁。

### 10.2 驱动选择矩阵

| 功能需求 | 推荐驱动 | 备选方案 |
|---------|---------|---------|
| 2.8~4.3寸显示 | `lcd.c` (MCU 接口) | - |
| 7寸及以上显示 | `ltdc.c` (RGB 接口) | - |
| 电阻触摸 | `touch.c` (XPT2046) | ADS7843 |
| 电容触摸 | `ft5206.c` / `gt9xxx.c` | - |
| 姿态传感 | `sh3001.c` + `st480mc.c` | - |
| 拍照/预览 | `ov5640.c` + `dcmi.c` | - |
| 音频播放 | `es8388.c` + SAI | - |
| 大容量缓冲 | `sdram.c` | - |
| 文件存储 | `norflash.c` + FATFS | SDMMC |

### 10.3 性能优化建议

1. **LCD 刷新**: 使用 `rcmd_xxx` 批量渲染，减少 `rcmd_flush()` 调用次数
2. **触摸采样**: 利用 `ft5206_scan()` 中的分频机制，降低 I2C 查询频率
3. **摄像头**: DCMI DMA 使用循环模式，配合帧中断处理
4. **音频**: SAI 使用 DMA 传输，避免 CPU 轮询
5. **SDRAM**: 关键数据使用 Cache，外设寄存器禁用 Cache

### 10.4 常见问题排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| LCD 白屏/花屏 | 初始化失败/时序错误 | 检查 `lcddev.id` 读取值 |
| 触摸无响应 | I2C 通信失败 | 检查 `tp_dev.init()` 返回值 |
| 摄像头黑屏 | SCCB 通信失败 | 检查 `ov5640_read_id()` |
| 音频无声 | ES8388 未初始化 | 检查 I2C 地址和寄存器写入 |
| SDRAM 数据错误 | 刷新频率不当 | 验证刷新率计算 |
| 栈溢出 | 任务栈太小 | 增大 `configMINIMAL_STACK_SIZE` |
