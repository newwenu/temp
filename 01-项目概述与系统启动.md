# FreeOS Desktop 技术手册 —— 第一部分：项目概述与系统启动

## 1. 项目概述

### 1.1 项目简介

**FreeOS Desktop** 是一个基于 FreeRTOS 实时操作系统的嵌入式桌面操作系统，运行在正点原子 ATK Apollo STM32H743 开发板上。项目采用安卓风格的桌面交互设计，支持多应用图标显示、触摸交互、应用生命周期管理，并集成了多种硬件外设，包括 IMU（惯性测量单元）、摄像头、USB 音频、文件系统等。

该系统的设计目标是在资源受限的嵌入式环境中提供一个现代化、可扩展的桌面应用框架，使开发者能够方便地添加和管理各类应用程序。

### 1.2 目标平台

| 属性 | 规格 |
|------|------|
| **开发板** | 正点原子 ATK Apollo STM32H743 |
| **处理器** | ARM Cortex-M7 |
| **主频** | 400 MHz |
| **开发环境** | Keil MDK-ARM |
| **工程文件** | `Projects/MDK-ARM/h743work.uvprojx` |
| **构建方式** | 仅通过 Keil IDE 构建，无 Makefile/CMake |
| **编译输出** | `Output/` 目录（.axf, .hex, .map 等） |

### 1.3 核心特性

- **安卓风格桌面交互**：图标化应用启动器，支持触摸滑动和点击
- **统一应用框架**：所有应用通过 `app_interface_t` 接口结构体实现标准化生命周期管理
- **静态内存分配**：所有 FreeRTOS 任务（包括空闲任务、定时器任务）均使用静态分配，消除动态堆分配风险
- **ISR 安全日志系统**：2KB 环形缓冲区，支持任务级和中断级双模式写入，带 LED 错误指示
- **渲染命令机制**：批量绘制命令缓冲，减少 LCD 刷新次数，提升渲染效率
- **多硬件集成**：IMU、摄像头、USB 音频、文件系统、SDRAM、NORFlash 等

## 3. 系统启动序列

### 3.1 启动流程概览

系统启动遵循严格的初始化顺序，分为**硬件初始化阶段**和**FreeRTOS 调度阶段**。整个流程在 [`User/main.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/User/main.c) 中实现。

```
┌─────────────────────────────────────────────────────────────┐
│                     系统启动流程                              │
├─────────────────────────────────────────────────────────────┤
│  1. sys_cache_enable()     → 开启 L1-Cache                  │
│  2. HAL_Init()             → 初始化 HAL 库                  │
│  3. sys_stm32_clock_init() → 设置系统时钟 400MHz            │
│  4. delay_init()           → 延时初始化                     │
│  5. usart_init()           → 串口初始化 (115200)            │
│  6. mpu_memory_protection()→ 内存保护区域配置               │
│  7. led_init()             → LED 初始化                     │
│  8. sdram_init()           → SDRAM 初始化                   │
│  9. lcd_init()             → LCD 初始化                     │
│ 10. startup_screen_show()  → 显示开机画面                   │
│ 11. key_init()             → 按键初始化                     │

│ 13. my_mem_init(x6)        → 6 个内存池初始化               │

│ 16. imu_init()             → IMU 姿态解算初始化             │
│ 17. st480mc_init()         → 磁力计初始化（调度器前）        │
│ 18. freertos_demo()        → FreeRTOS 入口                  │
│     ├── ring_log_init()    → 日志系统初始化                 │
│     ├── exfuns_init()      → 文件系统扩展初始化             │
│     ├── f_mount()          → 挂载文件系统                   │
│     ├── desktop_app_init() → 桌面应用初始化                 │
```

### 3.2 启动代码详解

#### 3.2.1 硬件初始化阶段

```c
int main(void)
{
    sys_cache_enable();                 /* 开启L1-Cache */
    HAL_Init();                         /* 初始化HAL库 */
    sys_stm32_clock_init(160, 5, 2, 4); /* 设置时钟, 400Mhz */
    delay_init(400);                    /* 延时初始化 */
    usart_init(115200);                 /* 串口初始化 */
    mpu_memory_protection();            /* 保护相关存储区域 */
    led_init();                         /* 初始化LED */
    sdram_init();                       /* 初始化SDRAM */
    lcd_init();                         /* 初始化LCD */
    startup_screen_show();              /* 显示开机画面（仅使用lcd基础绘图，不依赖字库） */
    key_init();                         /* 初始化KEY */
    norflash_init();                    /* 初始化NORFlash */
    my_mem_init(SRAMIN);                /* 初始化内部内存池(AXI) */
    my_mem_init(SRAMEX);                /* 初始化外部内存池(SDRAM) */
    my_mem_init(SRAM12);                /* 初始化SRAM12内存池(SRAM1+SRAM2) */
    my_mem_init(SRAM4);                 /* 初始化SRAM4内存池(SRAM4) */
    my_mem_init(SRAMDTCM);              /* 初始化DTCM内存池(DTCM) */
    my_mem_init(SRAMITCM);              /* 初始化ITCM内存池(ITCM) */
    
    while (fonts_init())                /* 初始化字库 */
    {
        lcd_show_string(30, 110, 200, 16, 16, "Font Error!", RED);
        delay_ms(200);
        lcd_fill(30, 110, 230, 126, WHITE);
        delay_ms(200);
    }
    
    /* SH3001/IMU初始化需要在FreeRTOS启动之前完成 */
    if (sh3001_init() == SH3001_TRUE)
    {
        imu_init();
    }

    st480mc_init();                     /* ST480MC磁力计初始化（含delay_ms，需在调度器前） */

    freertos_demo();                    /* 调用FreeRTOS例程 */
}
```

#### 3.2.2 关键约束说明

**SH3001/IMU 和 ST480MC 初始化必须在 FreeRTOS 调度器启动前完成**，原因如下：

- `sh3001_init()` 和 `imu_init()` 内部使用 `delay_ms()` 延时约 270ms
- `st480mc_init()` 同样包含 `delay_ms()` 延时
- 如果在 FreeRTOS 调度器启动后使用这些延时函数，会导致 FreeRTOS 相关的时序问题甚至死锁
- 因此这些初始化被放在 `freertos_demo()` 调用之前

**OV5640/DCMI 初始化已移到 `camera_app.c`**：
- 相机硬件初始化不再在系统启动时执行
- 改为在进入相机应用时进行阻塞式初始化
- 减少系统启动时间，避免未使用相机时的资源浪费

#### 3.2.3 FreeRTOS 入口阶段

```c
void freertos_demo(void)
{
    /* 初始化日志后台任务和LED指示 */
    ring_log_init();

    /* 初始化并挂载 NOR Flash 文件系统 */
    exfuns_init();
    f_mount(fs[1], "1:", 1);

    /* 初始化桌面应用 */
    desktop_app_init();
    
    /* 创建开始任务 */
    StartTask_Handler = xTaskCreateStatic((TaskFunction_t   )start_task,
                                          (const char*      )"start_task",
                                          (uint32_t         )START_STK_SIZE,
                                          (void*            )NULL,
                                          (UBaseType_t      )START_TASK_PRIO,
                                          (StackType_t*     )StartTaskStack,
                                          (StaticTask_t*    )&StartTaskTCB);
    vTaskStartScheduler();
}
```

开始任务 `start_task` 在临界区内创建桌面任务和触摸任务，然后自我删除：

```c
void start_task(void *pvParameters)
{
    taskENTER_CRITICAL();           /* 进入临界区 */
    
    /* 创建桌面任务 */
    desktop_task_create();
    
    /* 创建触摸任务 */
    touch_task_create();
    
    /* 删除开始任务 */
    vTaskDelete(StartTask_Handler);
    StartTask_Handler = NULL;
    
    taskEXIT_CRITICAL();            /* 退出临界区 */
}
```

### 4.6 中断嵌套配置

```c
#define configPRIO_BITS                                 4
#define configLIBRARY_LOWEST_INTERRUPT_PRIORITY         15
#define configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY    5
#define configKERNEL_INTERRUPT_PRIORITY                 (15 << (8 - 4))  /* = 240 */
#define configMAX_SYSCALL_INTERRUPT_PRIORITY            (5 << (8 - 4))   /* = 80 */
```

- **中断优先级范围**：0（最高）~ 15（最低）
- **FreeRTOS 可管理的最高中断优先级**：5
- **高于优先级 5 的中断**：不能调用 FreeRTOS API（ISR-safe 函数除外）
- **低于等于优先级 5 的中断**：可以安全调用 FreeRTOS API

### 4.7 断言处理

```c
extern void vAssertCalled(const char *file, int line);
#define configASSERT( x ) if( ( x ) == 0 ) vAssertCalled( __FILE__, __LINE__ )
```

断言处理函数 `vAssertCalled` 区分任务上下文和中断上下文：

```c
void vAssertCalled(const char *file, int line)
{
    volatile uint32_t icsr = *(volatile uint32_t *)0xE000ED04;
    uint32_t vectactive = icsr & 0xFF;
    const char *taskName = (xTaskGetSchedulerState() != taskSCHEDULER_NOT_STARTED) 
                           ? pcTaskGetName(NULL) : "none";

    taskDISABLE_INTERRUPTS();
    printf("ASSERT: %s:%d, task=%s, ICSR=0x%08lX, VECTACTIVE=%lu\n",
           file, line, taskName ? taskName : "NULL", icsr, vectactive);

    if (vectactive == 0)
    {
        /* 线程模式（任务上下文）：尝试只终止当前任务，让其余任务继续运行 */
        printf("ASSERT: task context, deleting self instead of panic\n");
        taskENABLE_INTERRUPTS();
        vTaskDelete(NULL);
        taskYIELD();
    }

    /* 中断上下文或 vTaskDelete 异常返回：仍视为致命错误，停机 */
    sys_panic();
}
```

**断言处理策略**：
- **任务上下文**（`vectactive == 0`）：仅终止当前任务，系统其余任务继续运行
- **中断上下文**：触发 `sys_panic()`，关闭中断，LED 红灯快闪指示致命错误

## 5. 静态内存分配

### 5.1 设计原则

项目采用**静态内存分配**作为核心设计原则，所有 FreeRTOS 任务（包括空闲任务、定时器任务）均使用静态分配的栈和控制块，消除动态堆分配带来的内存碎片和分配失败风险。

### 5.2 空闲任务与定时器任务的静态分配

```c
static StackType_t  IdleTaskStack[configMINIMAL_STACK_SIZE];        /* 空闲任务堆栈 */
static StaticTask_t IdleTaskTCB;                                    /* 空闲任务控制块 */
static StackType_t  TimerTaskStack[configTIMER_TASK_STACK_DEPTH];   /* 定时器任务堆栈 */
static StaticTask_t TimerTaskTCB;                                   /* 定时器任务控制块 */
```

### 5.3 静态分配钩子函数

FreeRTOS 要求实现 `vApplicationGetIdleTaskMemory()` 和 `vApplicationGetTimerTaskMemory()` 钩子函数来提供静态内存：

```c
void vApplicationGetIdleTaskMemory(StaticTask_t **ppxIdleTaskTCBBuffer, 
                                   StackType_t  **ppxIdleTaskStackBuffer, 
                                   uint32_t     *pulIdleTaskStackSize)
{
    *ppxIdleTaskTCBBuffer   = &IdleTaskTCB;
    *ppxIdleTaskStackBuffer = IdleTaskStack;
    *pulIdleTaskStackSize   = configMINIMAL_STACK_SIZE;
}

void vApplicationGetTimerTaskMemory(StaticTask_t **ppxTimerTaskTCBBuffer,
                                    StackType_t  **ppxTimerTaskStackBuffer,
                                    uint32_t     *pulTimerTaskStackSize)
{
    *ppxTimerTaskTCBBuffer   = &TimerTaskTCB;
    *ppxTimerTaskStackBuffer = TimerTaskStack;
    *pulTimerTaskStackSize   = configTIMER_TASK_STACK_DEPTH;
}
```

### 5.4 应用任务的静态分配

以开始任务为例：

```c
#define START_TASK_PRIO 1
#define START_STK_SIZE  128

static StackType_t StartTaskStack[START_STK_SIZE];
static StaticTask_t StartTaskTCB;
static TaskHandle_t StartTask_Handler = NULL;
```

创建任务时使用 `xTaskCreateStatic`：

```c
StartTask_Handler = xTaskCreateStatic((TaskFunction_t   )start_task,
                                      (const char*      )"start_task",
                                      (uint32_t         )START_STK_SIZE,
                                      (void*            )NULL,
                                      (UBaseType_t      )START_TASK_PRIO,
                                      (StackType_t*     )StartTaskStack,
                                      (StaticTask_t*    )&StartTaskTCB);
```

### 5.5 栈溢出检测

配置 `configCHECK_FOR_STACK_OVERFLOW = 2` 启用方法 2 栈溢出检测。当检测到栈溢出时，调用钩子函数：

```c
void vApplicationStackOverflowHook(TaskHandle_t xTask, char *pcTaskName)
{
    (void)xTask;
    (void)pcTaskName;
    sys_panic();    /* 关中断，LED快闪指示致命错误 */
}
```

栈溢出被视为致命错误，触发 `sys_panic()` 进行系统停机保护。

## 6. 内存池管理

### 6.1 内存池架构

系统实现了 **6 个独立内存池**，分别对应 STM32H743 的不同内存区域：

| 内存池标识 | 内存区域 | 说明 |
|-----------|---------|------|
| `SRAMIN` | AXI SRAM | 内部 AXI 总线 SRAM |
| `SRAMEX` | SDRAM | 外部 SDRAM |
| `SRAM12` | SRAM1 + SRAM2 | 内部 SRAM1 和 SRAM2 合并 |
| `SRAM4` | SRAM4 | 内部 SRAM4 |
| `SRAMDTCM` | DTCM | 数据紧密耦合内存 |
| `SRAMITCM` | ITCM | 指令紧密耦合内存 |

### 6.2 内存池初始化

在 `main()` 中依次初始化 6 个内存池：

```c
my_mem_init(SRAMIN);     /* 初始化内部内存池(AXI) */
my_mem_init(SRAMEX);     /* 初始化外部内存池(SDRAM) */
my_mem_init(SRAM12);     /* 初始化SRAM12内存池(SRAM1+SRAM2) */
my_mem_init(SRAM4);      /* 初始化SRAM4内存池(SRAM4) */
my_mem_init(SRAMDTCM);   /* 初始化DTCM内存池(DTCM) */
my_mem_init(SRAMITCM);   /* 初始化ITCM内存池(ITCM) */
```

内存池管理实现位于 [`Middlewares/MALLOC/malloc.c`](file:///g:/codeversion/h732worktask/基于FreeRTOS的作业实现/Middlewares/MALLOC/malloc.c)。

本章介绍了 FreeOS Desktop 的项目概况、目录结构、系统启动序列、FreeRTOS 配置以及静态内存分配机制。关键要点：

1. **系统启动分两个阶段**：硬件初始化阶段（在 `main()` 中）和 FreeRTOS 调度阶段（在 `freertos_demo()` 中）

5. **断言分级处理**：任务上下文仅终止当前任务，中断上下文触发系统停机
